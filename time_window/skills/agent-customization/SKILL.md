# Skill: Create SKILL.md for Agent Customization

## Purpose
Provide a reproducible, workspace-scoped procedure to extract a multi-step workflow from a conversation and turn it into a usable `SKILL.md` file that other agents and contributors can follow.

## Scope
- Intended: repository/workspace-scoped skill files.
- Not intended: system-level or UI automation outside the workspace.

## When to Use
- You have a multi-step conversational workflow (debugging, review, deployment, testing) and want to codify it as a reusable skill.

## Outcome
A complete `SKILL.md` saved in `skills/agent-customization/` containing:
- step-by-step process, decision points, quality criteria, clarifying questions, example prompts, and next-customizations.

## Workflow (Step-by-step)

1. Extract
   - Read the conversation and list the explicit steps followed by the user/agent.
   - Capture any branching logic (if X then Y; otherwise Z) and required inputs.

2. Identify Decision Points
   - Mark where human choices are required (scope, thresholds, post-checks).

3. Draft
   - Write a concise SKILL.md draft with: purpose, scope, step list, decision points, completion criteria, example prompts.

4. Save
   - Create file at `skills/agent-customization/SKILL.md` in the workspace.

5. Clarify
   - List ambiguous items as questions to the user (scope, examples, required outputs).

6. Iterate & Finalize
   - Incorporate user answers, finalize wording, add examples and usage notes.

## Decision Points (examples)
- Is the skill workspace-scoped or personal? (affects file location and access)
- Should the skill automate file edits or only generate guidance?
- Are there required artifacts (templates, config files) that must be present?

## Quality Criteria / Completion Checks
- Every step has at least one explicit input and expected output.
- Branching logic is documented with clear conditions.
- Includes 2–3 example prompts demonstrating typical usage.
- File saved in the workspace and linkable from agent docs.

## Example Prompts to Try
- "Create a SKILL.md that converts this conversation into a testing workflow." 
- "Generate a SKILL.md to standardize our model-training checklist for this repo."

## Clarifying Questions To Ask the User (template)
1. Should the skill be workspace-scoped or personal to your account?
2. What exact outputs should the skill produce when run? (file path, summary, checklist)
3. Any specific conventions to follow (filename, folder path, markdown style, required headers)?

## Suggested Next Customizations
- Add automated tests that validate the presence of required sections.
- Create a template folder `skills/templates/` with a canonical SKILL.md example.
- Add example-driven prompts for common variants (CI, debugging, code-review).
